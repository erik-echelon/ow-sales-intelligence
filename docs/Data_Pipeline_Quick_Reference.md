# Data Pipeline Quick Reference

**Purpose:** Step-by-step guide to regenerate all Streamlit app data from scratch

**Last Updated:** 2026-03-03

---

## Prerequisites

1. **Environment Variables:**
   ```bash
   export DATA_AXLE_API_KEY="your_key_here"
   export HUBSPOT_API_KEY="your_key_here"
   export ANTHROPIC_API_KEY="your_key_here"
   ```

2. **Python Environment:**
   ```bash
   uv pip install -e .  # or: pip install -e .
   ```

3. **Data Dependencies:**
   - `data/ow_industry_info/fct_churn_predictions.csv` (from dbt/data warehouse)
   - `data/ow_industry_info/fct_support_tickets.csv` (from dbt/data warehouse)
   - `data/ow_industry_info/metrics_by_naics.csv` (from dbt/data warehouse)
   - `data/config/exclusions.yaml` (industry exclusions)
   - `data/config/channels.yaml` (channel definitions)

## Pipeline Execution Steps

| Step | Epic | Script | Runtime | Command | Output Files | Purpose |
|------|------|--------|---------|---------|--------------|---------|
| **0** | **HubSpot Pull** | `hubspot_pull.py` | ~5 min | `python scripts/hubspot_pull.py --all` | `data/raw/hubspot_companies.csv`<br>`data/raw/hubspot_buildings.csv`<br>`data/raw/hubspot_contracts.csv`<br>`data/raw/hubspot_tickets.csv` | Pull current customer data from HubSpot CRM |
| **1** | **Epic 1** | `run_epic1_hq_discovery.py` | ~30 min | `python scripts/run_epic1_hq_discovery.py` | `data/processed/epic1_v4_<timestamp>/companies_hq_only.csv`<br>`data/processed/epic1_v4_<timestamp>/buildings_hq_only.csv` | Discover HQ companies from DataAxle API for target NAICS codes |
| **2** | **Epic 2** | `run_epic2_company_resolution.py` | ~10 min | `python scripts/run_epic2_company_resolution.py` | `data/processed/epic2_v5_<timestamp>/penetration_by_company.csv`<br>`data/processed/epic2_v5_<timestamp>/penetration_by_naics.csv`<br>`data/processed/entity_resolution_log.csv` | Match DataAxle prospects with HubSpot customers |
| **3** | **Epic 4 Prep** | `prepare_epic4_inputs.py` | ~1 min | `python scripts/prepare_epic4_inputs.py` | `data/processed/companies.csv` ⭐<br>`data/processed/buildings.csv` ⭐<br>`data/processed/penetration_by_company.csv` | Prepare standardized input files for scoring |
| **4a** | **Epic 4** | `analyze_support_tickets_semantic.py` | ~20 min | `python scripts/analyze_support_tickets_semantic.py` | `data/processed/customer_ticket_semantic_analysis.csv` | Semantic analysis of support tickets by customer |
| **4b** | **Epic 4** | `aggregate_naics_risk_metrics.py` | ~2 min | `python scripts/aggregate_naics_risk_metrics.py` | `data/processed/naics_risk_metrics.csv` | Aggregate churn + ticket health by NAICS |
| **4c** | **Epic 4** | `generate_naics_icp_scores.py` | ~35 min | `python scripts/generate_naics_icp_scores.py` | `data/scoring/naics_icp_fit_scores.csv` ⭐ | Claude AI scoring of industry ICP fit (308 NAICS codes) |
| **4d** | **Epic 4** | `generate_company_icp_scores_with_research.py` | ~3-4 hrs | `python scripts/generate_company_icp_scores_with_research.py` | `data/scoring/company_icp_scores_with_research.csv` ⭐ | Claude AI company scoring with web research (6K+ companies) |
| **4e** | **Epic 4** | `score_companies_final.py` | ~10 sec | `python scripts/score_companies_final.py` | `data/scoring/scored_companies_final.csv` ⭐<br>`data/scoring/naics_metrics.csv` ⭐ | Final weighted scores combining NAICS + Company |
| **5** | **Deploy** | `migrate_to_streamlit_repo.sh` | ~1 min | `bash migrate_to_streamlit_repo.sh` | Copies all app/ and data/ files to public repo | Sync all updates to public Streamlit Cloud repo |
| **6** | **Epic 5** | Launch Streamlit | Instant | `streamlit run app/streamlit_app.py` | N/A | Launch UI - loads all generated data files |

**⭐ = Required by Streamlit app**

---

## Total Pipeline Runtime

- **Fast path (no LLM):** ~1 hour (Steps 0-3 only)
- **Full pipeline:** ~5-6 hours (all steps)
- **Most expensive:** Step 4d (company ICP scoring with research) = 3-4 hours

---

## Key Output Files Required by Streamlit

The Streamlit app (`app/streamlit_app.py`) loads these files:

| File Path | Generated By | Required? | Purpose |
|-----------|--------------|-----------|---------|
| `data/raw/hubspot_companies.csv` | Step 0 | ✅ Yes | Customer company data |
| `data/raw/hubspot_buildings.csv` | Step 0 | ✅ Yes | Customer building locations |
| `data/processed/companies.csv` | Step 3 | ✅ Yes | All companies (prospects + customers) |
| `data/processed/buildings.csv` | Step 3 | ✅ Yes | All building locations |
| `data/processed/golden_buildings.csv` | Epic 2 (optional) | ⚠️ Optional | Enhanced buildings with HubSpot data |
| `data/processed/penetration_by_company.csv` | Step 2 | ⚠️ Optional | Customer penetration metrics |
| `data/processed/entity_resolution_log.csv` | Step 2 | ⚠️ Optional | Entity resolution diagnostics |
| `data/scoring/scored_companies_final.csv` | Step 4e | ✅ Yes | Final scored companies (main data) |
| `data/scoring/naics_metrics.csv` | Step 4e | ✅ Yes | NAICS attractiveness scores |
| `data/scoring/naics_icp_fit_scores.csv` | Step 4c | ✅ Yes | NAICS ICP fit details |
| `data/scoring/company_icp_scores_with_research.csv` | Step 4d | ✅ Yes | Company research details |
| `data/config/exclusions.yaml` | Manual | ⚠️ Optional | Industry exclusions |
| `data/config/channels.yaml` | Manual | ⚠️ Optional | Channel definitions |

## Incremental Updates

### Refresh Customer Data Only (Run Weekly)
```bash
# Re-pull HubSpot data
python scripts/hubspot_pull.py --all

# Re-run entity resolution
python scripts/run_epic2_company_resolution.py

# Regenerate standardized inputs
python scripts/prepare_epic4_inputs.py

# Recalculate final scores (uses cached NAICS/company scores)
python scripts/score_companies_final.py

# Copy to public repo and deploy
bash migrate_to_streamlit_repo.sh
cd ~/repos/ow-sales-intelligence
git add . && git commit -m "Weekly data refresh: $(date +%Y-%m-%d)" && git push
```
**Runtime:** ~15 minutes + deployment

### Refresh Prospect Data Only (Run Monthly)
```bash
# Re-run Epic 1 (new prospects from DataAxle)
python scripts/run_epic1_hq_discovery.py

# Re-run Epic 2 (match new prospects)
python scripts/run_epic2_company_resolution.py

# Regenerate standardized inputs
python scripts/prepare_epic4_inputs.py

# Re-score new companies
python scripts/generate_company_icp_scores_with_research.py

# Recalculate final scores
python scripts/score_companies_final.py

# Copy to public repo and deploy
bash migrate_to_streamlit_repo.sh
cd ~/repos/ow-sales-intelligence
git add . && git commit -m "Monthly prospect refresh: $(date +%Y-%m-%d)" && git push
```
**Runtime:** ~4 hours + deployment

### Refresh Scoring Only (Run as Needed)
```bash
# Re-score NAICS industries (if prompt changed)
python scripts/generate_naics_icp_scores.py

# Re-score companies (if prompt changed)
python scripts/generate_company_icp_scores_with_research.py

# Recalculate final scores
python scripts/score_companies_final.py

# Copy to public repo and deploy
bash migrate_to_streamlit_repo.sh
cd ~/repos/ow-sales-intelligence
git add . && git commit -m "Scoring updates: $(date +%Y-%m-%d)" && git push
```
**Runtime:** ~4 hours + deployment

## Troubleshooting

### App won't start - missing files
```bash
# Check which files are missing
ls data/raw/hubspot_*.csv
ls data/processed/companies.csv
ls data/scoring/scored_companies_final.csv

# If missing, run the appropriate pipeline step above
```

### Stale data in app
```bash
# Clear Streamlit cache (in browser)
Press 'C' key

# Or restart Streamlit
pkill -f streamlit
streamlit run app/streamlit_app.py
```

### API rate limits
- **DataAxle:** Default 200 calls/run (configurable with `--api-budget`)
- **Anthropic Claude:** No hard limit, but watch usage/costs
- **HubSpot:** Rate limited by HubSpot (handled automatically)
